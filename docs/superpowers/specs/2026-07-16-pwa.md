# Spec 7.1 — PWA (instalável + offline)

> Fase 7, item 7.1. Deixa o JurassicRun **instalável** (Android/desktop) e **jogável
> offline** via service worker com precache dos assets. Só camada de build/infra/app —
> `src/core/` **intocado** ⇒ determinismo **67 inalterado** (sem re-pin de goldens).

## Objetivo

- Manifesto Web App válido (nome, ícones, cores, orientação, display standalone) ⇒
  navegadores oferecem o prompt de instalação.
- Service worker que faz **precache** de todos os assets do build (JS/CSS/HTML/ícones,
  incluindo o chunk grande do Phaser ~1.4 MB) ⇒ a partida abre e roda **sem rede** após
  a primeira visita.
- Atualização automática do app quando um novo build é publicado (`autoUpdate`).

## Fora de escopo (outros itens da Fase 7)

- **Base path de subdiretório** (`vite base`) para GitHub Pages → **7.3**. O
  `vite-plugin-pwa` deriva `scope`/`start_url` do `base` do Vite automaticamente; ao
  setar `base` em 7.3, manifesto e SW se ajustam sem tocar esta spec.
- **Responsividade final / letterbox / safe-areas** → **7.2** (o `viewport-fit=cover` e
  `orientation` já entram aqui, mas a validação transversal é 7.2).
- **Deploy** (Pages/itch.io) → **7.3/7.4**.
- **Arte real dos ícones** → **Fase 8**. Aqui os ícones são **placeholders geométricos**
  (fundo sólido + triângulo, ecoando o dino cosmético do render), com asset-spec (REGRA 5)
  para substituição futura sem tocar código.

## Arquitetura (padrão puro × casca, como o resto do projeto)

Três peças:

### 1. Manifesto como módulo PURO testável — `src/pwa/manifest.ts`
Exporta o objeto de opções do PWA (`pwaOptions`) e o `manifest` (nome, `short_name`,
`description`, `theme_color`, `background_color`, `display: 'standalone'`,
`orientation: 'any'`, `icons[]`, `categories`). `vite.config.ts` (casca) importa e passa a
`VitePWA(pwaOptions)`. Isso torna o contrato do manifesto **unit-testável** sem rodar o
build (molde de `src/services/online/schema.ts` do 6.1). Nada de I/O no módulo.

`icons[]` referencia `icons/icon-192.png`, `icons/icon-512.png`, `icons/icon-maskable-512.png`
(caminhos **relativos**, sem barra inicial ⇒ corretos sob qualquer `base` de 7.3).

### 2. Gerador de ícones PURO — `src/pwa/icon.ts` + script `scripts/gen-icons.mjs`
Encoder PNG **puro** (só `node:zlib` + CRC32 manual — zero dep nova, no espírito
"sem custo, sem lib pesada"): `encodePng(width, height, rgbaPixels) → Uint8Array`
(assinatura PNG + IHDR + IDAT deflate de scanlines filtradas type 0 + IEND, CRC por chunk).
`renderIcon(size, { maskable }) → rgba` desenha o placeholder: fundo sólido
(`background_color`) + triângulo (cor de destaque). `maskable` encolhe o triângulo para a
**safe-zone de 80%** (evita corte em máscaras circulares/arredondadas).

`scripts/gen-icons.mjs` chama essas funções puras e escreve os 3 PNGs em `public/icons/`.
Roda via `npm run gen:icons`. **Os PNGs são comitados** (o build não depende de rodar o
script; é reprodutível e trocável em Fase 8). O `public/` inteiro é copiado pelo Vite para
`dist/`.

Ambas as funções (`encodePng`, `renderIcon`) são puras/determinísticas ⇒ testadas:
assinatura PNG válida, dimensões IHDR corretas, byte-determinismo.

### 3. Registro do SW — `injectRegister: 'auto'`
`registerType: 'autoUpdate'` + `injectRegister: 'auto'`: o plugin injeta o script de
registro no `index.html` do build; **nenhum código de app** (`main.tsx` intocado).
`devOptions.enabled: false` ⇒ SW não interfere em `npm run dev`. `workbox.globPatterns`
inclui `**/*.{js,css,html,png,svg,ico,woff2}`; `maximumFileSizeToCacheInBytes` elevado (~4 MB)
para caber o chunk do Phaser no precache; `navigateFallback` para `index.html` (SPA/rota
por signal, sem hash-routing ainda — Fase 7 posterior).

### 4. `index.html` (casca)
Adiciona `<meta name="theme-color">`, `<meta name="description">`, `<link rel="icon">` e
`<link rel="apple-touch-icon">` apontando para os ícones. O `<link rel="manifest">` é
**injetado pelo plugin** (não hardcodar). `lang="en"` já é ajustado em runtime pelo
`settingsService` (4.8) — mantido.

## Config do plugin (antes: `VitePWA({ disable: true })`)

```
VitePWA({
  registerType: 'autoUpdate',
  injectRegister: 'auto',
  includeAssets: ['icons/*.png'],
  manifest,                     // do módulo puro
  workbox: {
    globPatterns: ['**/*.{js,css,html,png,svg,ico,woff2}'],
    maximumFileSizeToCacheInBytes: 4 * 1024 * 1024,
    navigateFallback: 'index.html',
    cleanupOutdatedCaches: true,
  },
  devOptions: { enabled: false },
})
```

## Testes

- **`src/pwa/manifest.test.ts`** (unit, node): `manifest.name`/`short_name` não vazios;
  `display === 'standalone'`; existe ícone 192 e 512 `purpose any`; existe 512 `maskable`;
  `theme_color`/`background_color` são hex válidos; `pwaOptions.registerType === 'autoUpdate'`.
- **`src/pwa/icon.test.ts`** (unit, node): `encodePng` emite assinatura PNG correta
  (`89 50 4E 47 …`) e IHDR com largura/altura pedidas; `renderIcon(192)` e `(512)` retornam
  buffer do tamanho esperado; determinismo (mesma entrada ⇒ mesmos bytes).
- **`tests/pwa/icons-exist.test.ts`** (fs): os 3 PNGs existem em `public/icons/` e começam
  com a assinatura PNG (garante que o artefato comitado casa com o gerador).
- **Verificação de build** (manual, `verification-before-completion`): `npm run build` gera
  `dist/manifest.webmanifest`, `dist/sw.js`/`dist/workbox-*.js`, `dist/icons/*.png`, e o
  `index.html` do build contém o registro do SW. Opcional: `vite preview` + Lighthouse/DevTools
  confirmam "installable" e SW ativo (best-effort no ambiente headless).

## i18n

Nomes do manifesto (`name`/`short_name`/`description`) **não** passam por i18next — o
manifesto é estático por build (limitação da plataforma; localização de manifesto é um
recurso separado, fora de escopo). Sem strings de UI novas ⇒ REGRA 4 não se aplica. O nome
do app fica `"JurassicRun"` (marca, já na allowlist i18n).

## Determinismo / REGRAS

- REGRA 1 (determinismo): `src/core/` **não é tocado**. det 67 inalterado.
- REGRA 2 (arte desacoplada) + REGRA 5 (asset-spec): ícones são trocáveis; asset-spec em
  `docs/assets/specs/pwa-icon.md` + registro.
- REGRA 3 (performance): SW/manifesto são artefatos de build; zero trabalho por frame.
- REGRA 4 (i18n): sem strings de UI novas.

## Riscos / notas

- **Chunk do Phaser no precache**: o precache do workbox baixa tudo na 1ª visita (~1.5 MB
  total). Aceitável para um jogo instalável; é o preço do "jogável offline". `runtimeCaching`
  fino fica para depois se necessário.
- **`base` de 7.3**: não fixar `scope`/`start_url` absolutos aqui; deixar o plugin derivar
  do `vite base` (default `/` hoje) para 7.3 não quebrar.
- **HTTPS**: SW só ativa em HTTPS/localhost — garantido no Pages (7.3) e no `vite preview`.
