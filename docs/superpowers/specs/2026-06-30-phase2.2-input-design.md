# Spec — Fase 2.2: Input (flap + pausar/retomar)

> Data: 2026-06-30 · Fase 2 (vertical slice Endless) · Item 2.2 do
> `docs/roadmap/PHASE-02-endless-vertical-slice.md`.
> Modo autônomo (decisões técnicas tomadas seguindo `ARCHITECTURE.md` /
> `CLAUDE.md`; sem gate humano).

## Objetivo

Dar **input real** ao vertical slice: o pterodáctilo bate asas (flap) por **toque, clique ou
tecla**, amostrado **uma vez por step** da simulação (o loop de passo fixo já amostra a
`InputSource` 1×/step). E **pausar/retomar** a partida sem quebrar o passo fixo nem o
determinismo.

**Escopo estrito (só 2.2).** Não inclui: parallax (2.3), HUD/overlay textual (2.4), fluxo de
partida/seed aleatória e restart (2.5), Game Over (2.6), pooling/medição de fps (2.7). O cue
visual de pausa aqui é **não-textual** (escurecimento), porque rótulo "Pausado" é string de UI
e entra com o HUD/overlays (2.4/2.6) via i18next — evita violar a REGRA 4 nesta fatia.

## Regras inegociáveis aplicáveis

- **REGRA 1 (determinismo):** input vive **fora** de `src/core/`. O core continua puro: `step`
  recebe um `InputFrame` e faz a detecção de **borda de subida** (`input.flap && !lastFlap` ⇒
  um flap por pressão). 2.2 **não toca `src/core/`**. Pausar = **deixar de chamar `step`**;
  nunca mexe no estado da simulação.
- **REGRA 3 (performance):** `sample()` no caminho quente é **alocação-zero amortizada** (sem
  `new` por step; reusa o objeto `InputFrame`). DOM listeners não rodam por frame.
- **REGRA 4 (i18n):** nenhuma string visível hardcoded. 2.2 não introduz texto de UI (o cue de
  pausa é gráfico, não textual).

## Semântica do flap

O core faz **edge-detection**: segurar o botão dá **um** flap; para flapar de novo é preciso
soltar e pressionar (semântica Flappy). Portanto a `InputSource` reporta o **estado atual**
do botão (segurado ou não); a borda fica no core. Consequências de design:

- **Múltiplos inputs simultâneos** (ex.: dedo + tecla): rastrear um **conjunto** de fontes
  ativas (`Set<string>` de ids: `pointer:<id>`, `key:Space`, …). `held = size > 0`. Soltar uma
  fonte não cancela o flap se outra segue ativa. Set absorve **autorepeat** de `keydown` e
  `pointerdown` duplicado (re-adicionar o mesmo id é no-op).
- **Tap sub-frame (latch):** um toque que começa e termina **dentro de um mesmo frame** (entre
  dois `sample()`) teria `held=false` no próximo sample e **perderia** o flap. Para robustez,
  um **latch** marca "houve uma pressão desde o último sample"; `sample()` retorna
  `held || latched` e **limpa** o latch. Assim um tap rápido vira exatamente 1 flap (latch no
  sample em que é consumido; nos seguintes `held=false` ⇒ a borda do core não repete).

## Pausar/retomar

- **Estado de pausa** é da camada de render, não da simulação. `PauseController` expõe
  `paused`, `toggle()`, `pause()`, `resume()`.
- Quando `paused`, `GameScene.update` **não chama `loop.advance`** (a simulação congela) e
  **não acumula** tempo de pausa (como o frame é pulado inteiro, nada entra no acumulador; o
  `MAX_FRAME_TIME` do loop já protege o frame de retomada). O cenário é redesenhado **estático**
  + um **overlay escurecido** (sem texto).
- **Gatilhos:** tecla `P` ou `Escape` alterna pausa; **`blur` da janela auto-pausa** (não morre
  com a aba escondida). `focus` **não** auto-retoma (evita salto súbito) — retomar é explícito.
- **Anti-flap-fantasma:** ao **pausar**, limpar o estado de flap (held + latch). Assim uma
  pressão durante a pausa não dispara um flap na retomada; um ponteiro fisicamente ainda
  segurado só volta a flapar após soltar+pressionar.
- Teclas de pausa e de flap são **disjuntas** (`P`/`Escape` ≠ `Space`/`ArrowUp`/clique) ⇒ pausar
  nunca conta como flap.

## Arquitetura

`src/render/` continua dividido em **módulos puros e testáveis** (sem `phaser`, env node) e a
**casca** (DOM/Phaser, sem teste de unidade — espelha 2.1).

```
src/render/
  input.ts            # InputSource, NullInputSource (já existem)
                      # + FlapInputSource (puro): press/release/sample, Set + latch
                      # + PauseController (puro): paused/toggle/pause/resume (+ onPause hook)
  controls.ts         # casca DOM (não testada): bindGameControls(window, {flap, pause})
                      #   pointerdown/up/cancel + keydown/up → flap.press/release; P/Esc → toggle;
                      #   blur → pause. preventDefault em Space/ArrowUp.
  GameScene.ts        # gate de pausa no update + overlay escurecido (casca Phaser)
  game.ts             # createGame(parent, world, {input, pause}) injeta as duas peças
```

- **Puro/testado:** `FlapInputSource` (held/latch/multi-fonte, alocação-zero no `sample`) e
  `PauseController` (toggle/pause/resume + hook de limpeza de flap).
- **Casca/não testado:** `controls.ts` (listeners DOM finos que só encaminham para as peças
  puras) e o gate/overlay na `GameScene` (Phaser). Justificativa: igual a 2.1, a casca é fina o
  bastante para não exigir teste de unidade; a lógica testável está nas peças puras (env node,
  sem DOM).

### Constantes novas (`constants.ts`)
- `FLAP_KEYS = ['Space', 'ArrowUp']`, `PAUSE_KEYS = ['KeyP', 'Escape']` (usar `event.code`).
- `PAUSE_OVERLAY_COLOR`, `PAUSE_OVERLAY_ALPHA` (escurecimento).

### Integração `main.ts`
Cria `FlapInputSource` + `PauseController`, passa para `createGame`, e chama
`bindGameControls(window, { flap, pause })`. O `world` de demo segue (`endless:DEMO`).

## Plano de testes (Vitest, env node — sem DOM)

`tests/render/input.test.ts` (estende o existente):
- `NullInputSource` (mantido).
- `FlapInputSource`:
  - inicial: `sample()` ⇒ `{flap:false}`.
  - `press(a)` ⇒ próximos samples `flap:true` enquanto segurado; `release(a)` ⇒ `false`.
  - **edge/held:** segurar por vários samples ⇒ `flap:true` em todos (a borda é do core, não
    da fonte).
  - **multi-fonte:** `press(a); press(b); release(a)` ⇒ ainda `true`; `release(b)` ⇒ `false`.
  - **autorepeat:** `press(a); press(a)` (mesmo id) ⇒ um único release limpa.
  - **latch (tap sub-frame):** `press(a); release(a)` **sem** sample entre eles ⇒ o próximo
    `sample()` retorna `true` e o seguinte `false` (exatamente 1 flap).
  - **alocação-zero:** dois `sample()` retornam **o mesmo** objeto (reuso), e mutar não vaza.
    (ou: o objeto retornado é estável entre chamadas.)
  - **reset/anti-fantasma:** após `reset()`, held e latch zerados (`flap:false`), mesmo com id
    previamente pressionado.
- `PauseController`:
  - inicial `paused=false`; `toggle()` alterna; `pause()`/`resume()` idempotentes.
  - `onPause` hook é chamado ao entrar em pausa (liga em `FlapInputSource.reset` na integração).

Sem teste para `controls.ts`/`GameScene` (casca). Verificação visual manual (Playwright) do
fluxo flap + pausa, como em 2.1.

## Definição de pronto (2.2)

- Clique/toque/`Space`/`ArrowUp` fazem o pterodáctilo flapar (um flap por pressão).
- `P`/`Escape` pausa e retoma; `blur` auto-pausa; cenário congela e escurece na pausa.
- `npm run check` limpo, `npm test` verde (novos testes de input/pausa), determinismo intacto
  (core não tocado; bateria roda mesmo assim). Verificação visual confirmada.
