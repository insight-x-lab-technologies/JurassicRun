# Spec — Fase 2.1: Render Phaser sobre o core

> Data: 2026-06-30 · Fase 2 (vertical slice Endless) · Item 2.1 do
> `docs/roadmap/PHASE-02-endless-vertical-slice.md`.
> Modo autônomo (decisões técnicas tomadas seguindo `ARCHITECTURE.md` /
> `RENDERING-AND-ASSETS.md`; sem gate humano).

## Objetivo

Primeira camada de render. Phaser desenha a simulação determinística da Fase 1 num loop de
**passo fixo desacoplado do render**, com **interpolação entre steps**, lendo o `WorldState` e
desenhando pterodáctilo, obstáculos e pássaros-moeda **via manifesto de assets** (primitivas
geométricas). Prova a base técnica core↔render.

**Escopo estrito (só 2.1).** Não inclui: input real (2.2 — aqui só a interface + fonte nula),
parallax multicamadas (2.3), HUD (2.4), fluxo de partida/seed aleatória (2.5), Game Over (2.6),
pooling/medição de fps (2.7).

## Regras inegociáveis aplicáveis

- **REGRA 1 (determinismo):** o render **lê** o `WorldState` e **nunca** o altera fora do
  `step` do core. 2.1 **não toca `src/core/`**. O loop só chama `step(world, input)`.
- **REGRA 2 (arte desacoplada):** colisão/posição vêm da hitbox lógica; o visual vem do
  **manifesto**. Trocar geométrico↔PNG = editar o manifesto, não a lógica.
- **REGRA 3 (performance):** sem alocação por frame no caminho quente. A interpolação **não
  clona o mundo** por frame (ver "Interpolação").
- **REGRA 4 (i18n):** nenhuma string visível hardcoded. 2.1 não tem UI textual (HUD é 2.4); a
  seed de demonstração é dado técnico, não string de UI.
- **REGRA 5 (asset-specs):** todos os entities renderizados em 2.1 (`dino.default`, 4
  obstáculos, `bird.coin`) já têm spec em `docs/assets/specs/`. Nada novo a especificar.

## Arquitetura

`src/render/` é a camada de render (depende de `core`; nunca o contrário). Divide-se em
**módulos puros e testáveis** (sem `phaser`) e a **casca Phaser** (não testada por unidade):

```
src/render/
  constants.ts     # VIEW_WIDTH/HEIGHT, DINO_SCREEN_X, MAX_FRAME_TIME, cores de cenário
  interpolate.ts   # lerp(a,b,t)  — puro
  input.ts         # InputSource (interface) + NullInputSource — puro
  manifest.ts      # Renderable, ASSET_MANIFEST, renderableFor(), DINO_TYPE_ID — puro
  loop.ts          # FixedStepLoop — puro (importa core: step, FIXED_DT)
  GameScene.ts     # Phaser.Scene — desenha; importa phaser (NÃO testado por unidade)
  game.ts          # createGame(parent, world) → Phaser.Game — importa phaser
  index.ts         # reexports dos módulos puros (não reexporta GameScene/game p/ não puxar phaser)
```

Por que separar: Vitest roda em `node` (sem WebGL/DOM de canvas). Importar `phaser` num teste
quebra. Mantendo loop/interp/manifest/input livres de `phaser`, eles são testados de verdade; a
casca Phaser fica fina e é validada visualmente. `index.ts` **não** reexporta a casca Phaser,
para que importar `@render` em teste não puxe `phaser` transitivamente.

## Componentes

### `interpolate.ts`
`lerp(a, b, t) = a + (b - a) * t`. Puro, trivial, testado.

### `input.ts`
```ts
export interface InputSource { sample(): InputFrame; }   // InputFrame de @core/sim
export class NullInputSource implements InputSource { sample() { return { flap: false }; } }
```
2.1 só usa `NullInputSource`. 2.2 adiciona uma fonte de toque/tecla e a injeta no loop sem
mudar o loop.

### `manifest.ts` — manifesto de assets (REGRA 2)
```ts
export type Renderable =
  | { readonly kind: 'primitive'; readonly color: number; readonly shape?: 'hitbox' | 'triangle' }
  | { readonly kind: 'sprite'; readonly atlas: string; readonly frame?: string; readonly animation?: string };

export const DINO_TYPE_ID = 'dino.default';
export const ASSET_MANIFEST: Readonly<Record<string, Renderable>> = { /* ids → primitive+cor */ };
export function renderableFor(typeId: string): Renderable;  // fallback magenta p/ id desconhecido
```
- Ids: `dino.default` (`shape:'triangle'`, cor `0xcc5544` do asset-spec), `obstacle.tree`,
  `obstacle.vine`, `obstacle.boulder`, `obstacle.stalactite`, `bird.coin` (cores do
  `RENDERING-AND-ASSETS.md` / specs). Default `shape` = `'hitbox'` (desenha a própria hitbox).
- **Convenção da fase geométrica:** para `kind:'primitive'`, o render desenha a **geometria da
  hitbox** do entity na cor do manifesto (aabb→retângulo, circle→círculo, polygon→polígono);
  `shape:'triangle'` é exceção cosmética (dino) desenhada inscrita nos bounds da hitbox,
  apontando para +x. Quando vier arte, a entrada vira `kind:'sprite'` — só o manifesto muda.
- **Guarda de completude (teste):** todo `id` de `OBSTACLE_CATALOG` + `COLLECTIBLE_CATALOG` e o
  `DINO_TYPE_ID` devem ter entrada no manifesto. Falha ⇒ alguém adicionou um tipo sem visual.

### `loop.ts` — `FixedStepLoop` (passo fixo + interpolação)
Implementa o loop canônico de `ARCHITECTURE.md`:
```
advance(dtSeconds):
  dt = min(dtSeconds, MAX_FRAME_TIME)        # clamp anti spiral-of-death
  accumulator += dt
  while accumulator >= FIXED_DT:
    prev = (dinoX, dinoY)                     # snapshot ANTES do step (1 step atrás de curr)
    step(world, input.sample())               # core muta o mundo in-place
    accumulator -= FIXED_DT
  return stepsRun
alpha           = accumulator / FIXED_DT       # ∈ [0,1)
renderX/renderY = lerp(prev, curr, alpha)      # curr = world.pterodactyl.transform.position
```
- Estado de interpolação = **apenas a posição do dino** (`prevX/prevY` + dois números). Sem
  clone de `WorldState` por frame (REGRA 3).
- `input.sample()` é consultado **uma vez por step** (não por frame) — a amostragem por step é o
  que a sim espera; a detecção de borda do flap vive no `step` (`lastFlap`).
- Inicializa `prev = curr` (alpha 0, sem salto no 1º frame).

### `GameScene.ts` — casca Phaser (não testada por unidade)
- `extends Phaser.Scene`. `create()`: um `Phaser.GameObjects.Graphics` (desenho imediato; pooling
  é 2.7), guarda o `FixedStepLoop`.
- `update(_time, deltaMs)`: `loop.advance(deltaMs / 1000)`; depois **redesenha**:
  1. Câmera: `cameras.main.scrollX = loop.renderX - DINO_SCREEN_X`. (scrollY fixo: o mundo cabe
     na vertical.)
  2. Fundo simples (céu + faixas de chão/teto) — placeholder; parallax real é 2.3.
  3. Obstáculos e coletáveis: para cada entity, `renderableFor(entity.tags[0])` → cor; desenha a
     hitbox na posição de mundo do entity (estáticos ⇒ sem interpolação por entity).
  4. Dino: desenha o `DINO_TYPE_ID` (triângulo) em `(loop.renderX, loop.renderY)`.
- `performance.now`/RAF do Phaser ficam no render (permitidos fora do core).

### `game.ts` — `createGame(parent, world)`
Monta `new Phaser.Game({ type: Phaser.AUTO, width: VIEW_WIDTH, height: VIEW_HEIGHT, parent,
backgroundColor, scale: { mode: Phaser.Scale.FIT, autoCenter: CENTER_BOTH }, scene: [GameScene] })`,
injetando o `world` e um `NullInputSource` na cena (via `scene.data`/registry ou construtor).

### Integração — `src/app/main.ts`
Após `i18n.init()`, em vez do shell Preact vazio, cria um mundo de demonstração
`createWorld({ seed: 'endless:DEMO' })` e chama `createGame('app', world)`. (O fluxo real de
partida/seed aleatória é 2.5; aqui é só para "ver na tela".) O shell Preact de telas continua
sendo Fase 4.

## Mapeamento de coordenadas

- Canvas lógico **320×180**; `VIEW_HEIGHT = WORLD_HEIGHT (180)` ⇒ **1 unidade de mundo = 1px**,
  `+y` para baixo (igual ao core). `Scale.FIT` letterboxa em qualquer tela (responsividade
  fina é Fase 7).
- Dino fixo em `DINO_SCREEN_X = 80` (≈¼ da largura); câmera scrolla em x. Chão em
  `y = worldHeight`, teto em `y = 0`.

## Fluxo de dados

```
RAF Phaser → GameScene.update(deltaMs)
   → FixedStepLoop.advance(dt)  → N× step(world, NullInput)   [core muta world]
   → GameScene desenha: câmera(renderX) + manifest(cor)×hitbox(pos) + dino(renderX,renderY)
```
Sentido único: `app → render → core`. `core` não conhece ninguém.

## Tratamento de erros / bordas

- **Spiral of death:** `dt` clampado em `MAX_FRAME_TIME = 0.25s` (≤15 steps/frame) — aba em
  segundo plano não trava o navegador.
- **Morte do dino:** `step` congela o mundo (`alive=false` ⇒ early-return). O loop continua
  rodando e desenha o estado congelado (Game Over é 2.6). Em 2.1, com input nulo, o dino cai e
  morre — esperado; prova o pipeline, não a jogabilidade.
- **Id sem manifesto:** `renderableFor` cai num primitivo magenta (visível, não quebra) — mas a
  guarda de completude impede isso em código commitado.

## Testes (Vitest, env node — sem phaser)

- `tests/render/interpolate.test.ts`: `lerp` (extremos, meio, t fora de [0,1]).
- `tests/render/input.test.ts`: `NullInputSource.sample()` sempre `{flap:false}`.
- `tests/render/manifest.test.ts`: `renderableFor` conhecido vs fallback; **completude** (todo id
  de `OBSTACLE_CATALOG`+`COLLECTIBLE_CATALOG`+`DINO_TYPE_ID` presente).
- `tests/render/loop.test.ts`:
  - `advance(1.0)` ⇒ 60 steps (`world.tick += 60`); acumulador resvala ~0.
  - Acúmulo fracionado: dois `advance(0.5/60)` ⇒ exatamente 1 step.
  - `alpha` ∈ [0,1); `renderX/Y` entre `prev` e `curr`.
  - Clamp: `advance(100)` ⇒ ≤ `ceil(MAX_FRAME_TIME/FIXED_DT)` steps.
  - `InputSource` consultado 1×/step (fonte contadora).
  - Interpolação inicial sem salto (`prev=curr`, alpha 0 ⇒ render == posição inicial).

`GameScene.ts`/`game.ts` não têm teste de unidade (exigiriam WebGL/browser); são finos e
validados manualmente. `npm run check` (tsc+eslint) cobre a compilação/tipos deles.
